🏢 On-Premises Zone

  • SQL Server 2012+ with Change Tracking or CDC enabled on target tables
  • Dedicated service account (read-only, CDC permissions only)
  • Port 1433 open inbound from Azure Virtual Network Gateway subnet
  • On-prem firewall allows outbound to Azure ExpressRoute / VPN endpoint

🔀 Transit VNet (Hub)

  • Separate, pre-existing VNet — do not reuse the Databricks VNet as hub
  • Azure Virtual Network Gateway: ExpressRoute or VPN type
  • Optional Azure Firewall / NVA for traffic inspection and egress control
  • NAT Gateway required for new VNets after March 31, 2026 (stable egress IP)

⚡ Databricks VNet (Spoke)

  • VNet injection mandatory — Databricks cannot reach on-prem from default deployment
  • Two dedicated subnets: host (/26+) and container (/26+) — not shared with other services
  • VNet peered to Transit VNet with "Use Remote Gateways" + "Allow Gateway Transit"
  • User-Defined Routes added to route control plane traffic to internet, on-prem via gateway

🏆 Data Destination

  • Lakeflow Connect Ingestion Gateway runs in Databricks serverless compute — no extra VMs
  • SQL queries from Databricks clusters travel on-prem via the private path, never public internet
  • Unity Catalog streaming tables in ADLS Gen2 — Private Endpoint recommended for storage too
  • Downstream DLT Silver/Gold tables exposed to Power BI via Databricks SQL private endpoint
Choose Your On-Premises Connection Method
Azure · Recommended

🔵 Azure ExpressRoute

Dedicated private circuit between your on-prem network and Azure — bypasses the public internet entirely. Provisioned through a telco/ISP partner. Low latency, high throughput, SLA-backed.

✅ No public internet✅ SLA-backed✅ Consistent latency✅ Up to 10 Gbps
⚠ Higher cost⚠ Longer setup time (weeks)
Both · Acceptable

🟠 Site-to-Site VPN

IPSec VPN tunnel over the internet between your on-prem VPN device and Azure VPN Gateway. Supported for Lakeflow Connect if bandwidth is sufficient. Lower cost, faster to set up.

✅ Lower cost✅ Days to provision✅ Good for dev/test
⚠ Internet-dependent⚠ Variable latency⚠ Max ~1.25 Gbps
Both · Best Practice

🟢 ExpressRoute + VPN Failover

Production best practice: primary ExpressRoute circuit for all data traffic, Site-to-Site VPN as failover path. Azure Traffic Manager or BGP routing handles automatic failover if ExpressRoute goes down.

✅ High availability✅ Auto-failover✅ Zero data pipeline downtime
⚠ Highest cost

Breaking Change — March 31, 2026: New Azure VNets Require Explicit Outbound Connectivity
After March 31, 2026, new Azure VNets default to private configuration without outbound internet access. New Databricks workspaces created in new VNets require an explicit outbound connectivity method — Azure recommends attaching a NAT Gateway to the Databricks host and container subnets. Existing workspaces and VNets are not affected. When creating new infrastructure for this project, always attach a NAT Gateway to both Databricks subnets.
