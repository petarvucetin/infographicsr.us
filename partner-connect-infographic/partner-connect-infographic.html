Security & Compliance — Built In

  • No inbound firewall rules required
  • Outbound-only agent (port 443 / TLS 1.3)
  • Credentials stored in partner secret vault
  • No VPN or private link needed
  • OAuth service principal (no passwords)
  • Unity Catalog enforces row/column-level access
  • All data encrypted at rest (AES-256)
  • SOC 2 Type II certified partners

🗄️ On-Prem Setup

Install the Agent — a small background service on any Windows/Linux host that can reach your SQL Server
SQL Auth or Windows Auth — agent uses read-only DB credentials; stored locally in the agent config, never uploaded
Firewall — open port 443 outbound only. No inbound rules. DMZ or behind-NAT is fine.
CDC or batch — enable SQL Server Agent and CDC on your DB for real-time; otherwise batch polling works with no DB changes

🔗 Partner Connect Setup

Open Partner Connect in Databricks workspace sidebar → choose your partner (e.g. Fivetran)
One-click OAuth — Databricks auto-creates a service principal and grants least-privilege access to your catalog
Token auto-rotates — no manual credential management; Partner Connect handles token refresh
Destination pre-filled — Unity Catalog, schema, and storage location are auto-configured. You just pick tables.

🧱 Databricks Landing

Delta Lake tables — data lands as Delta with full ACID, time travel, and schema evolution
Unity Catalog — govern who sees what with column masking and row filters, no extra tooling
Auto schema drift — if SQL Server adds a column, Delta automatically evolves the schema
Query anywhere — SQL Warehouse, notebooks, or Power BI DirectQuery against the same Delta tables
