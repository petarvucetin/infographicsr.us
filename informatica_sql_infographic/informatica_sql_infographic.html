The 5-Step ETL Process
🔌
STEP 1
Connect
Configure ODBC / JDBC connections to both SQL Server instances in the Informatica repository.
📥
STEP 2
Extract
Source Qualifier reads tables, views, or SQL overrides from SQL Server A into the pipeline.
🔄
STEP 3
Transform + PII Mask
Apply Filter, Aggregator, Joiner, Lookup & Expressions. PII scrubbing is applied here — masking, redacting, or encrypting sensitive columns before load.
📤
STEP 4
Load
Target definition writes cleansed, transformed rows into SQL Server B using Insert / Update / Upsert.
📊
STEP 5
Monitor
Workflow Monitor tracks session logs, row counts, errors, and performance metrics in real time.

Masking Techniques Available
🔤
Redaction
Replace characters with *** or X — preserves field length
🔀
Substitution
Swap real values with realistic fake ones from a lookup dictionary
🔐
Encryption
AES encrypt field values — reversible with authorized key only
🎫
Tokenization
Replace values with unique tokens; original stored securely in vault
🔁
Shuffling
Randomly re-assign real values across rows — stats preserved
📏
Truncation
Keep only non-identifying prefix/suffix (e.g. ZIP, last-4 CC)
Informatica Tools for PII Handling
🛡️
Data Masking Transformation
Built-in PowerCenter transformation. Supports substitution, key masking, expression masking, and special-format masking for SSN, CC, phone, email, and dates.
🔍
Informatica CLAIRE™ AI
Automatically detects PII columns using ML-based data classification — flags SSNs, emails, names, and financial data across source schemas before mapping.
⚙️
Dynamic Data Masking (DDM)
Intercepts SQL queries at the proxy layer — masks results in real time based on user role without altering source data. Ideal for shared environments.
🔑
Expression Transformation
Write custom IIF / REGEXP_REPLACE / SUBSTR expressions to manually mask or hash any column using Informatica's built-in function library.
📋
Data Quality (IDQ)
Profiles data to discover PII patterns, validates formats, and applies business rules — works inline with PowerCenter or IICS pipelines.
📜
Audit & Lineage
Informatica Catalog tracks which fields were masked, by which rule, and in which run — providing full audit trail for GDPR / HIPAA / PCI-DSS compliance.
🔒 Where masking is applied: PII scrubbing occurs inside the Transform stage — after extraction from SQL Server A but before any row is written to SQL Server B. Source data is never modified.
Best Practices
⚡
Use Bulk Mode
Enable bulk loading on the target session to bypass row-by-row inserts and maximise throughput.
🗂️
Stage First
Load raw data into a staging schema before applying business logic to isolate failures cleanly.
🔑
Incremental Loads
Use high-watermark columns (created_at, modified_date) to process only changed rows each run.
🛡️
Error Handling
Configure reject files and pre/post SQL rollback statements to keep target tables consistent.
🔒
Mask at Transform
Always apply PII masking before the target write. Never store raw PII in staging or reject files.
