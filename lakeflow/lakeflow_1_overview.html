What is Lakeflow Connect?

Lakeflow Connect is Databricks' built-in, fully managed ingestion service — it connects directly to on-premises SQL Server (and cloud DBs) and replicates data continuously into Unity Catalog streaming tables using Change Data Capture or Change Tracking.

Unlike ADF or Fivetran, there is no external pipeline tool. Lakeflow Connect lives inside the Databricks platform — same workspace, same Unity Catalog, same serverless compute, one governance plane.

Key difference from ADF approach: No ADLS Gen2 landing zone to manage, no ADF pipelines to build. Lakeflow Connect handles snapshot + incremental CDC natively. Data lands straight into Unity Catalog streaming tables.

Zero External Tools

Entirely within Databricks — no ADF, no Fivetran, no SHIR VMs. Configure via Databricks UI or Declarative Automation Bundles (DABs).

♾️

Continuous + Incremental

First run ingests full snapshot. Every run after uses Change Tracking or CDC to ingest only changed rows — no full-table scans ever again.

🔁

Auto Schema Evolution

New columns added to SQL Server tables propagate automatically. Pipeline does not break on source schema changes — fully drift-tolerant.

🛡️

Fully Governed

Unity Catalog stores credentials, tracks data lineage, enforces RBAC, and provides audit logs for every row ingested — end to end.
