Lakeflow Connect — Step-by-Step Setup

8 steps to go from on-prem SQL Server to live Unity Catalog replication with Lakeflow Connect
3
of 4 · Setup
1
🗄️
Enable Change Tracking on SQL Server
Enable at the database and per-table level. Required for tables with primary keys (lowest overhead). SQL Server 2012+.
-- Enable on database ALTER DATABASE YourDB SET CHANGE_TRACKING = ON (CHANGE_RETENTION = 2 DAYS, AUTO_CLEANUP = ON); -- Enable on each table ALTER TABLE dbo.Orders ENABLE CHANGE_TRACKING WITH (TRACK_COLUMNS_UPDATED = ON);
SQL Server 2012+
2
💾
Enable CDC (Tables Without PKs)
Enable CDC for tables that have no primary key. Captures all column-level changes. Requires SQL Agent running.
-- Enable CDC on database EXEC sys.sp_cdc_enable_db; -- Enable on table (no PK) EXEC sys.sp_cdc_enable_table @source_schema = N'dbo', @source_name = N'Events', @role_name = NULL; -- Verify SELECT * FROM cdc.change_tables;
⚠ If both CT & CDC enabled on same table, Change Tracking takes precedence in Lakeflow Connect.
3
👤
Create Dedicated SQL Server Service Account
Create a dedicated SQL login for Databricks. Least-privilege access — read-only on the replicated database only.
-- Create login CREATE LOGIN databricks_lf WITH PASSWORD = 'Strong!Pass123'; -- Map to DB user USE YourDB; CREATE USER databricks_lf FOR LOGIN databricks_lf; -- Grant minimum required roles EXEC sp_addrolemember 'db_datareader', 'databricks_lf';
📌 Databricks recommends using an account with access ONLY to the replicated database — not sysadmin or db_owner.
4
🔐
Set Up Network Connectivity
On-prem SQL Server must be reachable from Databricks serverless compute. Choose your tunnel type.
Azure: Azure ExpressRoute (recommended) or Site-to-Site VPN. Route traffic to the Databricks control plane endpoint.
AWS: AWS Direct Connect or Site-to-Site VPN. Ensure port 1433 (SQL Server) is open from Databricks VPC/VNet.
⚠ VPN is supported only if bandwidth is sufficient for the expected data volume. Use ExpressRoute/Direct Connect for production.
Network / Firewall
5
🔗
Create Unity Catalog Connection
Store SQL Server credentials in a Unity Catalog Connection object. This is the secure credential store used by the Gateway.
CREATE CONNECTION sql_server_prod TYPE sqlserver OPTIONS ( host 'your-sqlserver.internal', port '1433', user 'databricks_lf', password secret('scope','sql_pass'), database 'YourDB' );
📌 Credentials stored encrypted in UC metastore. Supports username/password, OAuth (Entra ID), or M2M service principal auth.
6
⚡
Configure the Ingestion Gateway
Create the Gateway definition — it tells Lakeflow Connect where to extract data and where to stage it.
-- Via Databricks UI or DABs bundle: gateway: source_connection: sql_server_prod staging_catalog: main staging_schema: main_staging tables: - source_schema: dbo source_table: Orders - source_schema: dbo source_table: Customers
⚠ Staging catalog cannot be a foreign catalog — must be a native Unity Catalog catalog in your workspace.
7
🔄
Create the Ingestion Pipeline
Define the Ingestion Pipeline — destination catalog, schema, tables, and schedule. Lakeflow Connect auto-creates the Databricks Job.
pipeline: gateway: sql_server_gateway destination_catalog: main destination_schema: silver schedule: "*/15 * * * *" # every 15 min tables: - source_schema: dbo source_table: Orders destination_table: orders - source_schema: dbo source_table: Customers destination_table: customers
Serverless Compute
8
📊
Verify, Monitor & Connect BI
Check pipeline health, validate row counts match source, then connect Power BI or Tableau to Gold tables via Databricks SQL.
-- Check data freshness SELECT MAX(_commit_timestamp) AS last_updated FROM main.silver.orders; -- Row count validation SELECT COUNT(*) FROM main.silver.orders; -- Monitor in UC system tables SELECT * FROM system.lakeflow .ingestion_pipeline_events ORDER BY timestamp DESC LIMIT 20;
Power BI / Tableau

🟣 Deploy via DABs (Recommended)

  • Use Declarative Automation Bundles (Databricks CLI) to define Gateway + Pipeline as code
  • Version-control in Git — dev / staging / prod environments in the same bundle
  • Run databricks bundle deploy to provision or update the pipeline across workspaces
  • CI/CD integration: auto-deploy on PR merge via GitHub Actions or Azure DevOps

🔴 Required Databricks Permissions

  • CREATE CONNECTION on the Unity Catalog metastore (or use existing connection)
  • USE CATALOG + CREATE SCHEMA on both staging and destination catalogs
  • CREATE VOLUME on the staging schema (auto-managed by Lakeflow Connect)
  • Minimum 8-core worker node configuration for Gateway performance

🔵 Key Limitations to Know

  • Must connect to the primary SQL Server instance — read replicas / Always On secondaries not supported
  • Staging catalog cannot be a foreign catalog — must be native UC catalog
  • SQL Server 2012 SP1 CU3+ required for CDC; SQL Server 2012+ for Change Tracking
  • Enterprise Edition required for CDC on SQL Server versions before 2016
