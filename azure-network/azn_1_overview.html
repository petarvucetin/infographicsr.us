What does "private-only" mean?

No public-internet path to the Databricks workspace or the cluster workers exists. The only way in is through a VNet you own — reached from on-prem over VPN/ExpressRoute, or from another peered VNet.

Every Azure PaaS service the workspace touches (ADLS, Key Vault, SQL, etc.) is reached through a Private Endpoint — a NIC in your VNet that maps to the service. Public endpoints on those PaaS services are disabled.

Three locks to close: 1) Workspace Public Network Access = Disabled. 2) VNet-injected workers with Secure Cluster Connectivity — no public IPs. 3) Storage & secrets reached only via Private Endpoints with their public firewall closed.

🚫

Zero Public Exposure

No public IPs on workers. Workspace public access disabled. PaaS firewalls closed. Every internet reach is outbound-only through Azure Firewall.

🔒

Private Link Everywhere

Each Azure service a workspace touches (storage, KV, SQL, workspace API) gets a Private Endpoint in your VNet. Resolved via Private DNS zones.

🛣️

Explicit Egress

UDRs force all 0.0.0.0/0 through a central Azure Firewall. Application rules allowlist the specific FQDNs Databricks needs — nothing else.

🏗️

Customer-Owned VNet

Databricks deployed with VNet injection into a VNet you own. Peerable, NSG-able, UDR-able — not Databricks-managed.
